Spring Boot Auto-Configuration

Spring Boot’s Auto-Configuration simplifies application setup by automatically configuring components based on the classpath, beans, and properties. It eliminates the need for extensive manual configuration, making development faster and reducing boilerplate code.

Auto-configuration works through @EnableAutoConfiguration, which scans dependencies and provides sensible defaults. Developers can override these defaults using properties in application.yml or application.properties, or by defining custom beans.

Commonly Used Auto-Configuration Classes
  • DataSourceAutoConfiguration - Configures a DataSource bean automatically if a database driver (e.g., HikariCP) is on the classpath.
  • JpaRepositoriesAutoConfiguration - Enables Spring Data JPA repositories when Spring Data JPA is detected.
  • JacksonAutoConfiguration - Configures Jackson JSON processing, including ObjectMapper.
  • WebMvcAutoConfiguration - Sets up Spring MVC with DispatcherServlet, ViewResolvers, and MessageConverters.
  • SecurityAutoConfiguration - Configures Spring Security defaults, including a basic authentication mechanism.
  • CacheAutoConfiguration - Enables caching support when a caching library (e.g., EhCache, Redis) is present.
  • RabbitAutoConfiguration - Configures RabbitMQ integration if spring-rabbit is found in dependencies.
  • KafkaAutoConfiguration - Enables Apache Kafka messaging support if the spring-kafka library is included.
  • ActuatorAutoConfiguration - Configures Spring Boot Actuator endpoints for monitoring (health, metrics, etc.).
  • TaskExecutionAutoConfiguration - Configures asynchronous execution and a thread pool for @Async tasks.
Disabling Auto-Configuration

Spring Boot allows disabling specific auto-configurations using spring.autoconfigure.exclude in application properties or using the @EnableAutoConfiguration(exclude = ...) annotation.

Example: Disabling AWS CloudWatch Metrics Auto-Configuration in Dev
  1. Using application-dev.properties
    Add the following to your application-dev.properties file:

    spring.autoconfigure.exclude=io.micrometer.cloudwatch2.CloudWatchConfig, io.micrometer.cloudwatch2.CloudWatchMeterRegistry

  2. Using @EnableAutoConfiguration
    in Java You can also disable CloudWatch auto-configuration at the class level:

    @SpringBootApplication(exclude = {
      io.micrometer.cloudwatch2.CloudWatchConfig.class,
      io.micrometer.cloudwatch2.CloudWatchMeterRegistry.class
    })
      public class MyApplication {
      public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
      }
    }

    This ensures that CloudWatch metrics will not be configured when the application starts.
